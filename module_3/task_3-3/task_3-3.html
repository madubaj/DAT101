1. What is the difference between invoking and creating a function?

    2. Creating a function doesn't execute it, only defines it. The code inside a function is only executed when the function is invoked.


  2. Explain the syntax regarding semicolons and code blocks.

    3. Semicolons are used to end or separate statements in javascript to prevent potential errors.
    Code blocks are containers for multiple lines of code. Used with let or const, variables inside the block code is limited to the block in which they are defined. Which means that they are only accessible within that block.
    Code blocks are commonly used in functions, loops and if-else statements:

    • Code block inside a for loop contains one statement which is executed for each iteration of the loop. Multiple statements can be grouped into one block statement in the for loop.
    • In functions the code blocks contains blocks of code that executes when the function is called.
    • A code block in a conditional are used as the body of the loop, and contain statements that executes when the condition is met.


  3. What happens when we try to store values from a function when it doesn't return any?

    4. Output will be "undefined".


  4. How do we separate parameters from each other when creating a function?

    5. Parameters are separated with commas when creating a function.


  5. Explain how parameter names are related during the invocation and creation of a function.

    6. When creating a function, you give it parameters. These stores the values the function will use. When calling the function, you write in the real values the parameters will hold, called arguments. These arguments (the real values) are assigned to the parameters, and during the invocation the parameters receive the values of the arguments.


  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. The parameter is undefined because the function is called with missing arguments, meaning it's called with less parameters than declared. These missing values are then set to undefined.


  7. Explain what happens when we use recursive functions.

    8. When we use recursive functions, the function calls itself. Each time this happens, the function works on a subproblem until a base case is reached which then allows the recursion to stop.
